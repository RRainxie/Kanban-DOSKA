# Kanban doska(board) — система управления задачами

## 1. Общие сведения

**Название проекта:** Kanban Board

**Стейкхолдеры:**
- Заказчик: преподаватель вуза
- Команда разработки: студент (1 человек)
- Лица, принимающие решения: преподаватель

**Краткое описание:**
Веб-приложение для управления задачами в формате канбан-доски. Позволяет создавать задачи, перемещать их между колонками (Бэклог, В работе, Готово, Принято, Заблокировано) и удалять. Данные сохраняются в localStorage браузера. Проект предназначен для учебных целей и демонстрации навыков работы с HTML, CSS и JavaScript.

**Статус и дата:**
Версия 1.0, 26.05.2026

---

## 2. Контекст и цели

**Текущая проблема:**
Отсутствие наглядного инструмента для отслеживания прогресса по учебным задачам. Студенты теряют задачи, забывают дедлайны, не видят общую картину проекта.

**Цель:**
Разработать канбан-доску с базовым функционалом:
- создание задач
- перемещение между 5 колонками
- удаление задач
- визуальный лимит на колонку "В работе" (3 задачи)
- сохранение данных в браузере

**Анти-цели (Non-Goals):**
- Авторизация и регистрация
- Серверная часть (бэкенд)
- База данных (всё хранится в браузере)
- Drag-and-drop (перемещение только кнопками)
- Мобильное приложение
- Telegram/SMS уведомления
- Комментарии и вложения к задачам

---

Серверная часть отсутствует. Всё работает на стороне клиента.

**Описание компонентов:**

| Компонент | Функция |
|-----------|---------|
| HTML | Структура страницы (колонки, формы, кнопки) |
| CSS | Визуальное оформление (цвета, отступы, рамки) |
| JavaScript | Логика: добавление, перемещение, удаление задач, сохранение/загрузка из localStorage |
| localStorage | Постоянное хранение задач между сеансами |

**Потоки данных:**

1. Пользователь вводит текст задачи → нажимает "+" → задача добавляется в Бэклог → обновляется отображение → данные сохраняются в localStorage
2. Пользователь нажимает кнопку перемещения → задача удаляется из текущей колонки → добавляется в целевую → проверка лимита (3 задачи в работе) → перерисовка доски → сохранение
3. Пользователь нажимает "✖" → задача удаляется → перерисовка → сохранение
4. При загрузке страницы → данные читаются из localStorage → отображается доска
